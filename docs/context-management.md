# 컨텍스트 관리

## 문제

현재 에이전트는 대화가 길어지면 컨텍스트 윈도우를 초과한다.
messages 배열이 무한정 증가하고, 압축/요약 메커니즘이 없다.

## 자동 압축 (Compaction)

```
메시지 30개 초과 시:
  messages[0] = system prompt (유지)
  messages[1..N-10] → LLM으로 요약 → 1개 메시지로 대체
  messages[N-10..N] = 최근 10개 (유지)
```

### 압축 과정

1. 압축 대상 메시지 추출 (system prompt와 최근 10개 제외)
2. 별도 LLM 호출로 요약 생성 (저비용 모델 사용)
3. 요약 메시지를 system prompt 다음에 삽입
4. 원본 메시지 제거

### 보존 우선순위

| 우선순위 | 내용 | 이유 |
|----------|------|------|
| 1 | system prompt | 항상 필요 |
| 2 | 최근 10턴 | 현재 작업 컨텍스트 |
| 3 | 도구 실행 결과 요약 | 이전 행동의 결과 |
| 4 | 사용자 핵심 지시 | 방향성 유지 |
| 5 | 일반 대화 | 압축 대상 |

## System Reminder

N턴(기본 10턴)마다 시스템 메시지 주입:

```
현재 상태:
- 서버: xgen.x2bee.com (연결됨)
- 활성 도구: xgen_workflow_*, grep, bash
- 진행 중 작업: 워크플로우 "Mumu" 실행
- 대화 턴: 25/∞
```

## 서브에이전트를 통한 컨텍스트 분리

복잡한 작업은 서브에이전트에게 위임:
- 서브에이전트는 부모의 대화 히스토리 없이 새 컨텍스트로 시작
- 최종 결과만 부모에게 반환 (도구 결과로)
- 부모의 컨텍스트를 오염시키지 않음

```
부모 에이전트: "온톨로지 빌드해줘"
  ↓
spawn_agent("온톨로지 빌드", {
  systemPrompt: "온톨로지 전문가...",
  tools: [xgen_graph_*, bash],
  maxTurns: 10,
})
  ↓
서브에이전트: 독립 컨텍스트에서 작업 수행
  ↓
결과만 부모에게 반환: "온톨로지 빌드 완료. 트리플 1,234개 생성."
```
