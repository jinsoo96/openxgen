# 샌드박스 검증 환경

## 핵심 개념

샌드박스는 **정적 표시 패널이 아니라 실행 환경**이다.
사용자가 질문하면 LLM이 도구를 호출하고, 테스트하고, 결과를 보여주는 **라이브 환경**.

> "환경만 줘도 에이전트가 알아서 한다"
> 노드를 연결하는 게 아니라, 에이전트에게 도구와 인터프리터를 준다.

## 구조

```
┌──────────────┬──────────────────────────────┐
│   목록       │   샌드박스                     │
│   (탭별)     │   LLM이 자유롭게 실행          │
│              │   - API 테스트 → 결과          │
│              │   - 코드 실행 → 출력           │
│              │   - 도구 호출 → 응답           │
│              │   - 워크플로우 구조 탐색         │
├──────────────┴──────────────────────────────┤
│   채팅                                       │
│   ❯ "이 도구 테스트해봐" → LLM이 API 호출    │
│   ❯ "워크플로우 6번 실행해" → 실행 + 결과    │
│   ❯ "노드 구조 보여줘" → 파라미터/포트 표시   │
└──────────────────────────────────────────────┘
```

## 샌드박스 동작 원리

### 1. 사용자 입력 → LLM → 도구 호출 → 결과

```
사용자: "이 도구 API 테스트해봐"
   ↓
LLM (에이전트 루프):
  1. tool_search("tool") → 도구 스키마 로드
  2. xgen_tool_store() → 도구 목록
  3. apiTest({ url: "...", method: "GET" }) → HTTP 호출
   ↓
샌드박스 패널: 테스트 결과 표시
채팅 패널: "✓ 200 OK — 응답 정상"
```

### 2. 직접 실행 (Enter/단축키)

```
도구 탭에서 항목 선택 → Enter
   ↓
apiTest 직접 호출 (LLM 거치지 않음)
   ↓
샌드박스: 결과 표시
```

### 3. 하이브리드

채팅 입력은 LLM 에이전트 루프를 거치고,
단축키(Enter, t, u, d)는 직접 API 호출.

## 하네스 엔지니어링 관점

### Progressive Disclosure in Sandbox

에이전트가 샌드박스에서 작업할 때도 모든 도구를 한번에 주지 않는다:
1. 사용자가 "도구 테스트" → tool_search("tool") → 도구 관련 도구만 로드
2. 사용자가 "워크플로우 실행" → tool_search("workflow") → 워크플로우 도구만 로드
3. 필요한 것만 점진적으로 로드

### Hooks in Sandbox

- PreToolUse: 위험한 작업 차단 (rm -rf, 프로덕션 API 변경 등)
- PostToolUse: 결과를 샌드박스 패널에 표시, 채팅에 요약 추가

### 컨텍스트 분리

샌드박스 내 에이전트는 **독립 컨텍스트**로 실행:
- 대시보드 상태(탭, 선택 항목)를 시스템 프롬프트에 주입
- 이전 채팅 히스토리는 요약만 포함
- 샌드박스 결과는 채팅에 1줄 요약으로 추가

## 구현 계층

```
Layer 1: 직접 실행 (현재)
  - Enter → API 직접 호출 → 결과 표시
  - 빠르지만 LLM 판단 없음

Layer 2: 에이전트 루프 (목표)
  - 채팅 입력 → streamChat → tool calling 루프
  - LLM이 어떤 도구를 쓸지 판단
  - 결과를 해석하고 다음 행동 결정

Layer 3: 멀티 에이전트 (장기)
  - 샌드박스 전용 에이전트 (테스트/검증 특화)
  - 메인 에이전트가 위임 → 결과만 수신
```

## 검증 플로우

```
도구 생성 → 샌드박스에서 테스트 → 결과 확인 → 스토어 등록
프롬프트 작성 → LLM으로 테스트 → 결과 확인 → 저장
MCP 서버 → 세션 생성 → 도구 목록 → 호출 테스트 → 검증 완료
```

모든 과정을 대시보드 안에서, 채팅으로, 끊김 없이.
