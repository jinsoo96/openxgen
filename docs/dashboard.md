# 대시보드 설계

## 현재 문제

- 탭 선택 → 항목 선택 → Enter → 결과 — 단계가 많음
- 질문 입력과 결과가 분리됨 (입력은 하단, 결과는 오른쪽)
- 채팅 히스토리 없음

│              │                                  │

## 키 동작

### 공통
- `↑↓`: 목록 이동 (왼쪽 패널)
- `Enter`: 선택 항목 상세 (오른쪽 상단에 표시)
- `1-6`: 탭 전환
- `Tab/Shift+Tab`: 탭 순환
- `r`: 새로고침
- `q`: 종료

### 입력
- 아무 글자 입력 → 입력 바 활성화 (별도 모드 전환 불필요)
- `Enter`: 입력 실행 (현재 탭 컨텍스트에 따라)
  - 워크플로우 탭: 선택된 워크플로우로 질문 실행
  - 도구 탭: API 테스트
  - MCP 탭: MCP 도구 호출
- `Esc`: 입력 취소, 목록으로 복귀

### 탭별 단축키
- `c`: 생성 (도구/프롬프트/MCP 세션)
- `t`: 테스트
- `u`: 스토어 등록
- `e`: 수정
- `d`: 삭제
- `v`: 버전

## 채팅→결과 흐름

1. 사용자가 워크플로우 탭에서 항목 선택 (▸ 표시)
2. 바로 텍스트 입력 시작 → 입력 바에 표시
3. Enter → 선택된 워크플로우로 실행
4. 오른쪽 하단에 실행 결과 스트리밍 표시
5. 결과 히스토리 유지 (스크롤 가능)
